## Build Order

You are working on a large monorepo with many interconnected packages. When a developer makes a change, a set of build tasks must run — for example: compiling, linting, and testing.

Given a list of tasks and a list of dependencies where each pair `(task, prerequisite)` means the prerequisite must complete before the task can start, write a function that returns a valid execution order for all tasks.

If a circular dependency exists (e.g. A requires B and B requires A), return an empty list — the build would be stuck forever.

---

### Example

```
Tasks: ["linter", "compiler", "test_suite", "bundler"]

Dependencies:
  ("compiler",  "linter")    # must lint before compiling
  ("test_suite","compiler")  # must compile before testing
  ("bundler",   "compiler")  # must compile before bundling

Output: ["linter", "compiler", "test_suite", "bundler"]
Note: "test_suite" and "bundler" are interchangeable at the end.
```

---

### Constraints

- Task names are unique strings
- Dependencies are given as a list of `(task, prerequisite)` pairs
- Return any valid topological ordering
- Return `[]` if a cycle is detected
